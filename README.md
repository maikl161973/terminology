Django-приложение для управления справочниками терминологии с REST API.
(тестовое задание)


## Технологии

- Python 3.10+
- Django 5.2
- Django REST Framework
- SQLite
- drf-yasg (Swagger документация)


## API

### 1.Аутентификация и права доступа

API защищено токенной аутентификацией. 
Получить токен аутентификации:
`POST /api-token-auth/`
`Body: {"username": "ваш_username", "password": "ваш_пароль"}`

Использовать токен в заголовках запросов:
   Authorization: Token <ваш_токен>


### 2.Ролевая модель

Доступ к API ограничен пользователям, принадлежащим к группе с кодом роли из справочника. 
Код роли доступа для проверки по справочнику указывается в настройках проекта.


### 3. Получение списка справочников

`GET /api/refbooks/?date=YYYY-MM-DD`

Параметры:
- `date` (опционально) – дата в формате ГГГГ-ММ-ДД. Если указана, возвращаются только 
справочники, у которых есть версии с датой начала действия ≤ указанной даты.

Пример:

  http://localhost:8000/api/refbooks/?date=2026-02-01

Внимание: без аутентификации и настройки прав не сработает
Использовать токен в заголовках запросов:
   Authorization: Token <ваш_токен>


### 3. Получение элементов заданного справочника

`GET /api/refbooks/<код_справочника>/elements/?version=<версия>`

Параметры:
- `version` (опционально) – версия справочника. Если не указана, используются элементы текущей версии .


Пример:
  http://localhost:8000/api/refbooks/MED2/elements/?version=1.2

Внимание: без аутентификации и настройки прав не сработает
Использовать токен в заголовках запросов:
   Authorization: Token <ваш_токен>



### 4. Проверка элемента справочника

`GET /api/refbooks/MED1/check_element/?code=HR&value=Хирург&version=1.1`

Параметры:
- `code` – код элемента (обязательно)
- `value` – значение элемента (обязательно)
- `version` (опционально) – версия справочника. Если не указана, проверяется текущая версия.


Пример:
  http://localhost:8000/api/refbooks/SPEC/check_element/?code=1&value=Терапевт&version=1.0

Внимание: без аутентификации и настройки прав не сработает
Использовать токен в заголовках запросов:
   Authorization: Token <ваш_токен>
