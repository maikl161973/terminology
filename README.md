Django-приложение для управления справочниками терминологии с REST API.

## Технологии

- Python 3.10+
- Django 5.2
- Django REST Framework
- SQLite
- drf-yasg (Swagger документация)


## API

### 1. Получение списка справочников

`GET /api/refbooks/?date=YYYY-MM-DD`

Параметры:
- `date` (опционально) – дата в формате ГГГГ-ММ-ДД. Если указана, возвращаются только справочники, у которых есть версии с датой начала действия ≤ указанной даты.


Пример:

  http://localhost:8000/api/refbooks/?date=2026-02-01



### 2. Получение элементов заданного справочника

`GET /api/refbooks/<код_справочника>/elements/?version=<версия>`

Параметры:
- `version` (опционально) – версия справочника. Если не указана, используются элементы текущей версии .


Пример:
  http://localhost:8000/api/refbooks/SPEC/elements/?version=1.0


### 3. Проверка элемента справочника

`GET /api/refbooks/<код_справочника>/check_element/?code=<код_элемента>&value=<значение_элемента>&version=<версия>`

Параметры:
- `code` – код элемента (обязательно)
- `value` – значение элемента (обязательно)
- `version` (опционально) – версия справочника. Если не указана, проверяется текущая версия.


Пример:
  http://localhost:8000/api/refbooks/SPEC/check_element/?code=1&value=Терапевт&version=1.0
